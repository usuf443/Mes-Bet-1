# Обновления системы Zapretka - Регистрация и вход

## ✅ Что было изменено:

### 1. **Удалены предустановленные пользователи**
- Удалены 5 тестовых пользователей (Саня, Маша, Тимур, Александр, Иван)
- Система теперь работает с динамической регистрацией новых пользователей

### 2. **Реализована система регистрации**
- Экран регистрации с полями:
  - Никнейм (минимум 3 символа)
  - Пароль (минимум 4 символа)
  - Подтверждение пароля (проверка совпадения)
- Пользователи регистрируются в localStorage в виде объекта
- Каждый пользователь имеет профиль с аватаром, никнеймом, биографией и другой информацией

### 3. **Реализована система входа**
- Экран входа с полями:
  - Никнейм (обязательное поле)
  - Пароль (обязательное поле)
- Проверка учетных данных при входе
- Поддержка автоматического входа при обновлении страницы (сохранение в sessionStorage)

### 4. **Разделение чатов по пользователям**
- Каждый пользователь имеет свой набор сообщений в localStorage
- При входе загружаются сообщения только этого пользователя
- При выходе сообщения сохраняются в личное хранилище пользователя
- Пользователь видит только свои чаты и сообщения

### 5. **Структура файлов**

#### data.js (изменено)
- Добавлены функции регистрации:
  - `getRegisteredUsers()` - получить всех зарегистрированных пользователей
  - `registerUser(username, password)` - зарегистрировать нового пользователя
  - `verifyLogin(username, password)` - проверить учетные данные
  - `getUserData(username)` - получить данные пользователя
  - `updateUserData(username, userData)` - обновить данные пользователя

#### auth.js (новый/переделан)
- `handleLogin()` - обработка входа
- `handleRegister()` - обработка регистрации
- `switchToRegister()` - переключение на форму регистрации
- `switchToLogin()` - переключение на форму входа
- `checkAutoLogin()` - проверка автоматического входа при обновлении страницы
- `loadMessagesForUser(username)` - загрузка сообщений пользователя
- `saveMessagesForUser(username)` - сохранение сообщений пользователя

#### index.html (изменено)
- Экран входа переделан с двумя режимами (Вход/Регистрация)
- Форма регистрации с полем подтверждения пароля

#### script.js (изменено)
- Все сохранения сообщений обновлены на использование `saveMessagesForUser(me.name)`
- Все сохранения профиля обновлены на использование `updateUserData()`
- Удален старый код автоматического входа

## 📝 Как использовать:

### Регистрация
1. Нажмите "Создать аккаунт"
2. Введите никнейм (минимум 3 символа)
3. Введите пароль (минимум 4 символа)
4. Подтвердите пароль (должны совпадать)
5. Нажмите "Зарегистрироваться →"
6. После успешной регистрации будут перенаправлены на форму входа

### Вход
1. Введите никнейм, который использовали при регистрации
2. Введите пароль
3. Нажмите "Войти →"
4. После успешного входа загружается приложение со всеми вашими чатами

### Автоматический вход
- При обновлении страницы система автоматически входит с последним использованным аккаунтом
- Сеанс хранится в sessionStorage

## 🔒 Безопасность

⚠️ **Важно:** В текущей версии пароли хранятся в localStorage в открытом виде. Для продакшена необходимо:
- Реализовать хеширование паролей (bcrypt, Argon2)
- Использовать HTTPS для передачи данных
- Реализовать бэкэнд для безопасного хранения учетных данных

## 🗄️ Структура хранилища

### localStorage ключи:
- `zapretka_registered_users` - JSON с объектом всех зарегистрированных пользователей
  ```json
  {
    "username1": {
      "password": "password",
      "avatar": "url",
      "nickname": "name",
      "birthplace": "city",
      "birthyear": "2000",
      "bio": "text"
    }
  }
  ```
- `zapretka_messages_username` - сообщения для каждого пользователя

### sessionStorage ключи:
- `current_user` - имя текущего пользователя (для автоматического входа)
